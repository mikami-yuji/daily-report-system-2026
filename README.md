# 営業日報システム (Daily Report System)

エクセルベースの日報を、モダンなWebインターフェースで管理できるシステムです。Salesforceのような使いやすいUIで、日々の活動報告やデザイン依頼の管理を効率化します。

## ✨ 特徴

- 📊 **エクセル連携**: 既存のエクセルファイル（`.xlsm`）を直接読み書き。関数やマクロを保持したまま運用可能。
- ☁️ **Salesforce風UI**: 直感的でプロフェッショナルなデザイン。サイドバーナビゲーションやカード型レイアウトを採用。
- 🔍 **高度なデザイン検索**:
    - デザインNo.、得意先、デザイン名、種別でのクロス検索
    - 得意先ごとのデザイン種別動的フィルタリング
    - 進捗状況の可視化（色分けバッジ）
    - 関連する活動履歴の時系列表示
- 📈 **データ分析**: 月ごとの訪問数・電話数、重点顧客への活動状況をグラフで可視化。
- 📂 **ファイル管理**: ブラウザから直接エクセルファイルをアップロードして切り替え可能。
- 📱 **レスポンシブ**: PC・タブレット・スマホ対応。
- ⚡ **高速な動作**: Next.js 16 (App Router) と Tailwind CSS によるモダンな実装。

## 🚀 クイックスタート

### 前提条件
- Python 3.11以上
- Node.js 18以上
- npm または yarn

### セットアップ手順

#### 1. バックエンドの設定

```powershell
cd backend

# 設定ファイルを作成（初回のみ）
# エクセルファイルの保存先ディレクトリを指定
$json = '{"excel_dir": "C:\\Users\\YOUR_USERNAME\\.gemini\\antigravity\\daily-report-system"}'
[System.IO.File]::WriteAllText("$PWD\config.json", $json, (New-Object System.Text.UTF8Encoding $false))

# バックエンドを起動
python main.py
```

> **注意**: `config.json`の`excel_dir`は、エクセルファイル（`.xlsm`）を配置するディレクトリのパスを指定してください。

#### 2. フロントエンドの起動（別のターミナルで）

```powershell
cd frontend
npm run dev
```

#### 3. ブラウザでアクセス

`http://localhost:3000` を開く

### 簡単起動（設定済みの場合）

```powershell
# プロジェクトルートで実行
.\start_app.bat
```

このバッチファイルは、バックエンドとフロントエンドを同時に起動します。

## 🔧 トラブルシューティング

### API 500 エラーが発生する場合

バックエンドの`config.json`が正しく設定されているか確認してください。ネットワークパスではなく、ローカルの絶対パスを指定する必要があります。

### 無限ループで読み込み中になる場合

ブラウザのキャッシュをクリアしてページをリフレッシュしてください。最新版では修正済みです。

### ポートが使用中の場合

- バックエンド: デフォルトはポート8001
- フロントエンド: デフォルトはポート3000

他のプロセスが使用している場合は、該当プロセスを停止してください。
